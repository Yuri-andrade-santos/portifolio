Introdução

Este documento detalha as funcionalidades do Sistema de Gerenciamento Zabbix, uma aplicação de desktop desenvolvida em Python. O objetivo principal do sistema é fornecer uma interface gráfica amigável para interagir com a API do Zabbix, simplificando tarefas administrativas como a visualização de dados, criação de hosts em massa e a geração de relatórios de disponibilidade. A aplicação foi construída utilizando a biblioteca customtkinter para a interface gráfica e pyzabbix para a comunicação com o servidor Zabbix.

Autenticação de Usuário

O acesso ao sistema é protegido por uma tela de login. O usuário deve fornecer credenciais válidas (nome de usuário e senha) de uma conta existente no servidor Zabbix. A aplicação utiliza essas credenciais para se autenticar na API do Zabbix. Em caso de sucesso, o usuário é direcionado ao menu principal. Se a autenticação falhar, uma mensagem de erro é exibida, e o usuário pode tentar novamente. A sessão do usuário (usuário e senha) é mantida de forma segura enquanto a aplicação está em uso para executar as demais funções.

Painel Principal e Navegação

Após o login, o usuário é apresentado ao painel principal. Esta tela é dividida em duas seções principais:

  • Menu de Navegação: Uma barra lateral à esquerda que contém botões para acessar todas as funcionalidades principais do sistema.
  • Área de Conteúdo: Uma área à direita que exibe dinamicamente as informações solicitadas, como listas de hosts, templates ou outras interfaces de usuário.

Este design permite uma navegação intuitiva e um acesso rápido a todas as ferramentas disponíveis.

Funcionalidades Detalhadas

a. Visualização de Hosts e IDs (Hosts/ID)

Esta função permite ao usuário visualizar uma lista completa de todos os hosts monitorados no Zabbix. A aplicação busca e exibe o nome de cada host juntamente com seu hostid correspondente. Esta lista é apresentada em um formato rolável, facilitando a consulta de informações em ambientes com um grande número de dispositivos.

Visualização de Templates e IDs (Templates/ID)

Semelhante à função de visualização de hosts, esta seção exibe todos os templates disponíveis no servidor Zabbix, cada um com seu respectivo templateid. Esta funcionalidade é essencial para que o usuário saiba qual ID de template utilizar no arquivo CSV para a criação de novos hosts.

Criação de Hosts em Massa (Criar Hosts)

Esta é uma das funcionalidades centrais do sistema, permitindo a criação automatizada de múltiplos hosts a partir de um arquivo CSV. O usuário seleciona um arquivo .csv que deve conter as seguintes colunas: hostname, ip, groupid, templateid e port. A aplicação processa o arquivo linha por linha, utilizando a API do Zabbix para criar cada host com as configurações especificadas. Mensagens de sucesso ou erro são impressas no console durante o processo, e uma notificação final informa a conclusão da tarefa.

Geração de Relatório de Disponibilidade

O sistema possui uma poderosa ferramenta para gerar relatórios de disponibilidade de hosts. Nesta tela, o usuário pode definir um período específico (data e hora de início e fim) para a análise. O relatório calcula a porcentagem de tempo em que os hosts de um grupo específico (definido no código como "casa") estiveram ativos (UP) e inativos (DOWN).

Os resultados são exportados em dois formatos:

  • Arquivo CSV (availability_report.csv): Um relatório detalhado contendo os dados de disponibilidade de cada host, ideal para análises e armazenamento de registros.
  • Gráfico PNG (availability_report.png): Um gráfico de barras que compara visualmente as porcentagens de tempo UP e DOWN para cada host, facilitando a rápida identificação de problemas.

Logout e Encerramento de Sessão

Aplicaçã inclui uma funcionalidade de "Logout" que permite ao usuário encerrar sua sessão de forma segura. Ao clicar no botão de logout, a janela principal é fechada, e o usuário é redirecionado para a tela de login inicial. Isso garante que a conexão com a API do Zabbix seja finalizada, protegendo o acesso ao servidor.
